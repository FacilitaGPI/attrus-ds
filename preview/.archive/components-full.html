01Overview

The component library is the third layer of the system. Every component reads only from semantic tokens (Layer 2), never from raw values (Layer 1). Components are responsible for shape, behaviour, and accessibility — not for picking colours, sizes, or motion curves.

Stable

Production-ready. Used across the dashboard kit. Full spec page available.

New in 0.2

Just landed. Behaviour locked, spec page in progress.

Primitive

Headless building block. Used by other components, rarely on its own.

How to read each section

Each component below shows: a live canvas with the most representative variants, a quick anatomy table of variants/sizes/states, and a deep-link to its full specification page when one exists.
