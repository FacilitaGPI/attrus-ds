01Overview

A token is a named CSS custom property that holds a single design decision. Tokens flow through three layers — raw values become semantic roles, which components consume. Components never reach past their layer.

Layer 1 Raw palette --color-brand-primary-500: #003399 Hex codes, px values, font names. The literal source. Defined once, never referenced from a component.
Layer 2 Semantic role --color-foreground-accent: var(--color-brand-primary-500) Named by purpose, not by appearance. Maps a role to a raw value — the only level that may be re-themed.
Layer 3 Component color: var(--color-foreground-accent) Buttons, inputs, cards. Read semantic tokens; never read raw palette directly, never invent local values.

Principles

Reference, don't repeat

If two places need the same value, they reference the same token. Duplicating a hex code is a bug.

Semantic over raw

Components consume --color-foreground-accent, not --color-brand-primary-500. Roles can be remapped; raw values cannot.

No magic numbers

Padding of 11px? Border-radius of 7px? It doesn't exist. Round to the nearest scale step or add a token.

Light + inverse parity

Every foreground role on light has an --color-foreground-inverse-* twin for dark. Never improvise dark variants.

13How to consume

When you write a new component, the rule is simple: read semantic tokens, never raw values, never local hex codes. Below — the same button, two ways. Only one will survive a rebrand.

Do — semantic tokens
/* button.css */
.btn-primary {
  background:    var(--color-brand-primary-500);
  color:         var(--color-foreground-on-solid);
  border-radius: var(--radius-pill);
  padding:       var(--space-3) var(--space-6);
  font-size:     var(--fs-md);
  font-weight:   var(--w-semibold);
  box-shadow:    var(--shadow-sm);
  transition:    var(--transition-fast);
}
.btn-primary:focus-visible {
  box-shadow: var(--shadow-focus);
}
Don't — local values
/* button.css */
.btn-primary {
  background:    #003399;
  color:         #FFFFFF;
  border-radius: 999px;
  padding:       8px 20px;
  font-size:     13px;
  font-weight:   600;
  box-shadow:    0 1px 2px rgba(0,0,0,.08);
  transition:    all 150ms ease;
}
.btn-primary:focus-visible {
  outline: 2px solid #003399;
}

Rules

Always semantic

Components consume --color-foreground-default, --color-background-surface, --color-border-focus. Reaching for --color-neutral-800 directly is a code smell.

Raw is privileged

Raw palette tokens (--color-brand-primary-500, --color-success-100) are referenced only inside colors_and_type.css when defining semantic roles.

Add, don't override

Need a new shade? Add a token to the file. Never override a token's value inside a component stylesheet.

Audit before adding

Search the file before proposing a new token. The closest existing role is almost always the right answer.
