04Decoration — badges & counters

A row may carry one trailing decoration — a badge (qualitative: status / announcement) or a count (quantitative: how many). They answer different questions: a badge says "something here needs your attention / is new"; a count says "there are N of these". Never put both on the same row, and never more than one badge.

Badge types & function

Pill-shaped qualitative indicator (.badge + tone). Three tones, each with a distinct job. Content is 1 short word or 1–2 chars — a badge flags kind, not magnitude.

qualitative
Type Class Function — when to use Content
NEW .badge Announcement (brand). A newly shipped surface or capability. Promotional, not urgent. Retire after the feature is no longer new. 1 word — NEW, BETA
3 .badge.alert Alert (danger). Items that need action — failures, overdue, blocked. The only badge that may carry a small number, when that number is the urgency. 1–2 digits, or a dot
β .badge.warning Attention (warning). Caveat state — experimental, deprecated, needs review. Informs without demanding immediate action. 1 char / short — β, !

When NOT to use a group

Groups add a layer of nesting and a click cost. Use the table below to decide.

SituationUseWhy
1-2 sub-pagesTabs inside the pageNot worth the extra click. Tabs surface siblings without re-navigating.
3-6 related destinationsGroup / dropdownSweet spot. Collapse keeps the rail short; expand reveals all options.
7+ destinationsDedicated landing pageToo many for a dropdown to read at a glance. Build a "Hub" page with cards.
Mixed destinations + commandsSplit: nav-only group + page action barCommands belong in page headers, not the sidebar. Don't mix.
Dynamic count (user-created)Dedicated section with overflowPin the most-recent or pinned ones; offer a "See all" link to a full index.

08Motion

The sidebar has three motion responsibilities — width transition (collapse / expand), dropdown reveal, mobile drawer slide. Each uses a duration that fits its travel.

Collapse / expand rail width

The whole <aside> animates width from 244px → 60px (or back). Use --duration-base with --ease-standard — 200ms is long enough to feel intentional, short enough not to interrupt navigation.

200ms · ease-standard
aside { width: 244px; /* or 60px when .collapsed */ transition: width var(--duration-base) var(--ease-standard); }

Labels fade out independently (opacity 1 → 0 at --duration-fast) so they disappear before the rail finishes shrinking — otherwise they'd be sliced mid-letter. Section labels collapse to a 1-px hairline divider via the same width transition.

Dropdown reveal (group expand)

The children container uses grid-template-rows: 0fr → 1fr — the only height-animation technique that works without knowing the content's height in advance. Chevron rotates simultaneously.

200ms · ease-standard
.sb-children { overflow: hidden; display: grid; grid-template-rows: 0fr; /* collapsed */ transition: grid-template-rows var(--duration-base) var(--ease-standard); } .sb-children > .sb-children-inner { min-height: 0; } .sb-group[aria-expanded="true"] .sb-children { grid-template-rows: 1fr; /* expanded */ } .chev { transition: transform var(--duration-base) var(--ease-standard); } .sb-group[aria-expanded="true"] .chev { transform: rotate(90deg); }

Why grid-template-rows? Because height: auto can't be animated. Setting an explicit max-height forces an arbitrary cap that breaks long lists. The grid trick lets the browser measure intrinsic height and animate to it cleanly.

Mobile drawer — enter / exit

On viewports ≤ 900px the sidebar lives off-screen. It slides in from the left when the user opens the mobile menu, behind a blurred scrim that animates opacity in lockstep.

320ms · ease-standard
aside.mobile { position: fixed; top: 0; left: 0; bottom: 0; transform: translateX(-100%); /* hidden */ transition: transform var(--duration-slow) var(--ease-standard); box-shadow: var(--shadow-xl); /* when open */ } aside.mobile.is-open { transform: translateX(0); } .scrim { position: fixed; inset: 0; background: var(--color-scrim); backdrop-filter: blur(2px); opacity: 0; transition: opacity var(--duration-slow) var(--ease-standard); } .scrim.is-open { opacity: 1; }

Exit animation: when the scrim is clicked or the close affordance is tapped, the drawer translates back to -100% at the same 320ms duration. Both drawer and scrim animate together — they must arrive and leave in lockstep, or the scrim flashes empty for a frame.

Idle micro-transitions

Per-item state changes use --transition-fast (140ms). No padding-shifts, no icon nudges, no scale. Sidebars are heavily visited — anything slower than 150ms reads as drag.

10Don'ts

Six anti-patterns the sidebar consistently attracts. Each one breaks either muscle memory or readability.

Don't nest items deeper than one level

The sidebar is a top-level surface. Sub-pages live inside the content area (tabs, breadcrumbs, drawers). Never indent rows inside the sidebar.

Don't use coloured rows for status

The active row owns the brand-50 background. Don't paint other rows red/yellow/green to signal urgency — use a badge or count instead.

Don't reorder sections per tenant

The canonical order (product → company → integration) is muscle memory across the platform. Drag-and-drop reordering breaks new-employee onboarding for no win.

Don't combine badge and count

One trailing slot. Either say "3 things need attention" (badge) or "847 items total" (count). Stacking both reads as visual clutter.

Don't animate on idle → hover

The sidebar is heavily visited. Padding-shifts, glow effects and icon nudges read as motion sickness over a long session. Colour-only transitions, max 150ms.

Don't put primary CTAs in the sidebar

"New transaction", "Add counterparty" — these belong in the TopBar or page-level header. Sidebar items are destinations, not commands.
