Design system

Status pills

Compact, label-first indicators that tell the user the state of a record — a transaction, a counterparty, a compliance check. Pills don't drive action; they classify. They're scannable in tables, scannable in lists, and never replace a button.

01Anatomy

A pill is built from up to three parts inside a rounded container. The label is required; the dot and trailing icon are optional reinforcement. Order is fixed: dot → label → icon. Never break it.

Approved
1 Leading dot 6px circle · variant fg · purely decorative reinforcement.
2 Label 1–2 words · sentence case · the actual meaning. Always required.
3 Trailing icon Optional · 12px · reinforces the label, not a substitute for it.
4 Pill container radius-pill · 1px border · 20/24/28px tall depending on size.
PartPurposeTokens
ContainerRounded shape — defines the visual envelope of the status--radius-pill · height: 20/24/28

02Variants

Six semantic variants. Choose by what the status means, not by what colour you'd like. Variant gives the colour; the label gives the meaning. Both must agree.

Success .success

The record is in a final, healthy state. Transaction released, counterparty approved, document validated. End-state of a happy path.

success
Approved Released Available Received Settled

Use for

  • Final, irreversible positive state
  • Funds available, KYC complete
  • Audit-ready, fully settled

Don't use for

  • "In progress" — that's brand
  • Confirmation messages — that's a toast

Warning .warning

The record needs attention but isn't broken. Pending review, on hold, awaiting input. Action expected — but not urgent.

warning
Pending Under review Funds on hold To receive Awaiting docs

Use for

  • Transient state requiring follow-up
  • Manual review queues
  • Time-bound holds & checkpoints

Don't use for

  • Hard failures — those are danger
  • System errors — those are danger

Danger .danger

The record is in a failed, blocked, or rejected state. Compliance reproved, payment bounced, deadline missed. Negative end-state.

danger
Reproved Failed Bounced Blocked Overdue

Use for

  • Final negative outcome
  • Compliance / fraud rejections
  • Past SLA / deadline breaches

Don't use for

  • Destructive actions — those are buttons
  • Inline form errors — those are field help

Info .info

Neutral, non-evaluative metadata. Tags, channels, regions, types. Says what something is, not how it's doing.

info
PIX TED Boleto USD Tier 1

Use for

  • Categorical metadata (channel, type, currency)
  • Tags & filters
  • Read-only labels

Don't use for

  • Evaluative status — pick a real semantic
  • Counters — use a numeric badge

Neutral .neutral

Inactive or deprioritised states. Cancelled, archived, draft, identified. The record exists but isn't part of the active flow.

neutral
Cancelled Archived Draft Identified Inactive

Use for

  • Records out of the active pipeline
  • Soft-deleted, draft, paused
  • "Known but not assessed yet"

Don't use for

  • Default chip styling — that's info
  • Disabled UI — use opacity instead

Brand .brand

The record is currently being processed by Attrus systems. Specifically reserved for "in our hands" states. Use sparingly — it's a strong signal.

brand
Processing In analysis Routing Queued

Use for

  • Active processing inside Attrus
  • States the platform owns
  • Differentiate from external "pending"

Don't use for

  • Generic in-progress — that's warning
  • Marketing emphasis — pills aren't ads

05Styles

Three style modifiers control how loud the pill reads. Default (subtle) is correct for 95% of cases. Use .solid only for "look here" salience (a single critical row); .outline only on busy backgrounds.

Subtle · default

Tinted background, deep foreground, soft border. Reads quietly — works in tables and lists without competing with content.

Solid .solid

Filled with the variant's primary tint. Highest salience — reserve for one or two row callouts (e.g. blocked transactions you want the eye to find first).

Outline .outline

Transparent fill, full-strength border. Useful on photographic / busy backgrounds where any tinted fill clashes.

06States

Status pills are read-only by default. The interactive states below apply only when the pill is the control — typically a filter chip or a removable tag.

Read-only (base)

The default. No hover, no focus, no cursor change. The pill is a label, not a button.

default
Approved Pending Reproved

Interactive — filter chip .interactive

Add .interactive + aria-pressed to make the pill a toggle. Hover thickens the border; focus draws the standard ring; pressed state inverts to brand.

interactive

Triggers

  • :hover — border tightens to a stronger tint
  • :focus-visible — 4px focus ring (--shadow-focus)
  • aria-pressed=true — solid brand fill, white text

Required wiring

  • Use <button>, never <span>
  • Set aria-pressed on each chip
  • Group with role="group" + label

Removable — tag

Add a trailing .pill-x button when the user can clear the value (active filters, tokenised input).

removable
PIX USD Tier 1 Pending

Disabled aria-disabled="true"

Opacity 0.5 + not-allowed. Use only on interactive pills you cannot toggle right now (e.g. a filter that has zero matching results).

disabled

08On inverse

When pills sit on --color-background-inverse or --color-background-inverse-brand, default subtle pills wash out. Apply .on-inverse — bg becomes a translucent variant tint, fg lightens.

09Don'ts

Common pitfalls. Each undermines the contract that "variant + label = unambiguous status."

Don't ship a pill without a label. Colour alone fails for users with low colour vision and breaks screen-reader output.

Don't use a pill as a primary action. Pills classify; buttons act. If clicking changes state, it must be a button.

ReleasedPendingBouncedPIX

Don't use solid as default. Five solid pills in one row is visual noise. Subtle is the default; solid is the exception.

RELEASED · 2024-04-12 · R$ 184,200.00

Don't stuff metadata into the label. One status word. Domain data lives in adjacent table cells.

ApprovedReproved

Don't mismatch variant and meaning. Approved is success; Reproved is danger. The taxonomy is the contract.

Custom state

Don't invent new variants. If the existing six don't fit, the system is wrong — propose a new variant in the design system, don't hex-pick locally.
