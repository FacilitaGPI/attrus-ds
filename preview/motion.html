Design system

Motion

Snappy, never bouncy. One curve and five durations compose every animation in the system — from a 80ms tap echo to a 480ms page transition. Motion patterns (fade, slide, scale, pulse, shake) are the verbs you reach for; durations and easings are the adverbs that tune them.

02Easings

Three curves cover every case. --ease-standard is the default — an out-expo that lands soft. --ease-emphasis adds a touch of overshoot for entries. --ease-linear only for loading indicators where the user needs steady rate.

03Patterns

The animations the system uses. Each pattern is a composition of duration + easing + transform/opacity changes applied to specific surfaces. Hover any card to loop the motion.

Aa
fade

Opacity 0 → 1. The quietest entry. For tooltips, toast bodies, lazy-loaded content.

Aa
slide-up

translateY(20→0) + fade. For toasts (bottom origin), inline confirmations, sticky bars rising into view.

Aa
slide-down

translateY(-20→0) + fade. For dropdowns and popovers anchored to a trigger above.

Aa
slide-left (in)

translateX(+→0). Drawer entering from the right; back-button flow forward.

Aa
slide-right (in)

translateX(-→0). Drawer entering from the left; navigation flow back.

Aa
scale-in

scale(0.96 → 1) + fade. For modals, command palettes, focused overlays.

Aa
scale-down (out)

scale(1 → 0.94) + fade. The dismiss counterpart of scale-in.

Aa
pop

Scale 0.8 → 1.04 → 1, with --ease-emphasis. Use sparingly — confirmation badges, success states.

Aa
shake

Quick lateral oscillation for validation errors. 400ms total, do not loop.

●
pulse

Expanding ring of color-mix shadow. Live indicators, "recording" dots, pending payment markers.

spin

360° rotation at --ease-linear. Inline loaders only — never for "saving" status (use skeleton).

Aa
skeleton-shimmer

Gradient sweep across a placeholder shape. For content-loading states longer than 300ms.

05Reduced motion & accessibility

Vestibular sensitivity is real. Every animation in this system must short-circuit to its end state when prefers-reduced-motion: reduce is set.

@media (prefers-reduced-motion: reduce) { *, *::before, *::after { animation-duration: 0.01ms !important; animation-iteration-count: 1 !important; transition-duration: 0.01ms !important; scroll-behavior: auto !important; } }

The system applies this rule globally in dist/css/tokens.css. Don't override it per-component — even for "essential" animations like pulse loading. A static state always remains legible.

06Don'ts

Five anti-patterns that consistently appear in product mocks. Each one breaks the system's "snappy, never bouncy" rule.

Don't bounce

No spring physics, no overshoot beyond --ease-emphasis. The system reads as institutional — bounces feel like a children's app.

Don't animate everything

Color shifts, opacity, transforms — yes. Page reflows, layout changes, scroll positions — no. Animating the layout costs more than it pays.

Don't go over 480ms in-product

Only modals and drawers reach --duration-deliberate (640ms). Everything else above 480ms feels sluggish.

Don't invent easings

Three curves: standard, emphasis, linear. Custom cubic-beziers fragment the system's feel.

Don't loop without reason

Looping animations (pulse, spin, skeleton) draw attention. Only use them when their job is to indicate ongoing state — never decoration.
