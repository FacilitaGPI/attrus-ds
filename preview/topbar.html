Top bar

The application's top edge: primary action, global search, and a right-aligned cluster of session controls. One 32px control rhythm holds it all together.

01Anatomy

Left to right: the high-contrast primary action, then the search pill that grows to fill the gap, then a right-aligned cluster (.tb-spacer) of language, live status, utility icons and the user avatar. Every interactive element is a .tb-control at the same 32px height.

Full bar .topbar

52px tall, surface background, hairline bottom border. Mirrors the sidebar header height so logos align across the seam.

02Controls

Three forms of the same building block. Default for labelled actions, .icon for square icon-only buttons. The bar's single call to action is the canonical .btn-primary from the button system — not a top-bar-local style.

Control variants .tb-control

All 32px tall, same radius and hover. Hover any to see the border lift and background fill.

03Live indicator

A non-interactive control that signals an active real-time session. Same pill shell as the buttons so it sits in rhythm, but with a steady success dot and a monospace elapsed clock — never a hover state.

Session status .tb-live

Dot uses --color-success-500 with a soft 15% halo. The clock is mono --fs-xxs, muted.

04Mobile

Below 720px the bar collapses: the menu button and brand mark lead, the utility cluster drops to a single primary-action icon plus the avatar, and search shrinks to a glyph-first pill.

Compact bar

Menu toggle replaces the collapsed sidebar; secondary controls hide to protect the tap targets.
