Design system

Theming & products

Two orthogonal axes dress every screen. data-theme sets the ambient surface (light / dark); data-product sets the brand matiz (Enterprise / Business). They compose — a product needs no dark mode of its own beyond a few contrast exceptions.

color · gradient · inverse · shape

01The two axes

Keep them orthogonal and the matrix never explodes: a new product is a small matiz patch, not a second design system. The rule that keeps it cheap — theme owns surface + intensity; product owns matiz (color, accent, gradient). Focus ring, hero gradients and the radial glow all derive from the product's brand/accent tokens, so they re-skin for free.

Theme axis · data-theme

Surface & intensity

Shared by every product. Defined once in colors_and_type.css (:root + [data-theme="dark"]).

  • Backgrounds, foregrounds, borders
  • Feedback pairs (success/warning/danger/info)
  • Neutrals, shadows, tint alpha
Product axis · data-product

Brand matiz

Per product. Defined in themes/<id>.css; default (Enterprise) is :root.

  • Brand/accent color set + gradients (below)
  • Inverse layer (auto-tinted) + shape (roundness)
  • Dark: re-assert only the 3 foregrounds the dark block re-tints

04What a product may — and may not — redefine

The contract the drift linter enforces. Cross it and you no longer have a sub-brand; you have a fork.

May redefine

  • Brand color — the --color-brand-* set (solid, text, subtle, border, on-solid)
  • Accent — --color-accent-solid / -subtle + --color-foreground-accent / -highlight. Focus ring & radial glow follow accent automatically.
  • --color-foreground-brand (institutional text)
  • Gradients — auto-derive from --color-brand-solid; override --grad-hero / -brand / -brand-soft / -ink for a bespoke hero
  • Inverse layer — --color-background-inverse* auto-tint from --color-brand-solid (dark/premium panels follow the product)
  • Shape — --shape-control / -container / -overlay, each picking a step from the --radius-* scale (sharper vs rounder)
  • Logo asset (matiz)
  • Dark: only the foregrounds the dark block re-tints

May NOT redefine

  • Surface ramp — background-*, foreground-default/secondary/muted, border-*
  • Feedback — success / warning / danger / info
  • Structural gradients — sheen / surface-top / radial-glow / data-*
  • Neutrals, raw scales, spacing, shadows — and the --radius-* scale itself (shape only PICKS from it). Type/border-weight tokens are a planned future layer.
/* themes/<product>.css — the whole contract */
[data-product="business"] {
  --color-brand-solid: #1F6FD6;   /* matiz only — Business Blue */
  --color-brand-text:  #1657AD;   /* darkened so text passes AA */
  --color-foreground-brand: #1657AD;
  /* …subtle / border / hover derived from the primary… */
  /* accent stays shared Sky (not redefined) */
  --shape-control: var(--radius-md);   /* rounder than Enterprise */
  --shape-container: var(--radius-lg);
  --shape-overlay: var(--radius-xl);
}
[data-product="business"][data-theme="dark"] {
  /* only the institutional brand text lifts off ink */
  --color-foreground-brand: #9FC4FF;
}
