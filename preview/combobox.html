Design system

Combobox

Searchable picker used in two roles: listing (pick one item from a large dataset — internal accounts, counterparties, currencies) and smart searchbar (a global command-palette that orients the user across actions, records, docs and IDs).

01When to use

Two roles, same component. Choose based on what the user is trying to do.

Listing

The user is picking one item from a known dataset — a counterparty, an internal account, a currency, a country. Replaces a Select when options > ~10 or when fuzzy search matters.

Smart searchbar

The user is orienting themselves — they might want an action, a record, a doc, or to jump to a screen. The combobox guides: suggests, groups results, recognises IDs, recovers from typos.

Don't use for…

Multi-select (use multi-pick or chip-input), free-form text (use input), or yes/no decisions (use toggle / radio).

04Anatomy

Both modes share the same six pieces — only the body content swaps.

  1. Input row — leading icon (search / spinner), placeholder, optional clear (×) button. Always 36–40 px tall.
  2. Group label — uppercase tracking-wide eyebrow that introduces each section (Suggestions, Recent, Actions, API, FAQ, Result). Use --color-foreground-muted.
  3. Body content — chips (suggestions), list rows (results, recent searches), type-picker buttons (ID disambiguation), illustrations (empty / loading / error).
  4. Result-label slot — small eyebrow saying "Result (0)" or "Result" used in similar-match, no-results, ID search and error states. Separates "what we searched" from "what we got".
  5. Hint card — a translucent help block inside the body for did-you-mean copy, ID format reminders or contextual tips. Uses --color-background-subtle + a help-circle icon.
  6. Footer — sticky row with keyboard shortcuts: ↑↓ Navigate · ↵ Open · esc Close.

05Keyboard & accessibility

Combobox is keyboard-first; the smart searchbar is a global command (⌘K) — it must be operable without a mouse.

KeyBehaviourNotes
⌘/ctrl+KOpen the smart searchbar from anywhereHijacks; ignore inside text inputs.
↑ ↓Move highlight up/down between rowsWraps to top/bottom; skips group labels.
↵Activate the highlighted rowFor Actions → run command. For records → navigate. For docs → open external.
escClose the popover or clear the inputTwo-step on listing combobox: first esc clears input, second closes.
tabMove focus out of the comboboxDoes not select.
Screen readerrole="combobox" on the input, aria-controls pointing to the listbox, each row role="option" with aria-selected.Live-region announces result count: "5 results" / "No results".
