Two shapes of the same data. Both carry light values, dark-theme overrides, and resolved concrete values where a token references another.

Flat attrus-tokens.flat.json

One object keyed by token name → { value, resolved?, dark?, darkResolved? }. Best for lookups, scripts and diffing.

Download flat JSON

Nested attrus-tokens.json

Grouped by category, each leaf { value, darkValue? }. Style-Dictionary-style for build pipelines and codegen.

Download nested JSON
These files are generated from colors_and_type.css — don't hand-edit. Re-run the export whenever tokens change; the catalog's 334 tokens stat should match the file's tokenCount.

02Token shape

Authored value plus a resolved concrete value. color-mix() and gradients stay symbolic — they need a runtime that understands them.

"--color-background-canvas": {
  "value": "var(--color-neutral-25)",   // authored (may reference another token)
  "resolved": "#FAFBFD",                  // concrete, after following var() chains
  "dark": "var(--color-ink-deep)",        // dark-theme override (only when present)
  "darkResolved": "#0A1128"
}

04Consuming

TargetHow
CSS / webImport colors_and_type.css directly — the custom properties already ship. The JSON is for other targets.
Style DictionaryPoint a source at attrus-tokens.json; leaves use the { value } contract. Map darkValue to a dark mode.
iOS / Android / FlutterFeed attrus-tokens.flat.json to your generator; prefer resolved/darkResolved for concrete constants. Skip color-mix()/gradient values.
