Design system

Number Input

Numeric field paired with − / + accelerator buttons. Use for bounded quantities where the user nudges around an expected value: counts, percentages, days, seats. Not to be confused with Stepper — that's the multi-step flow indicator.

01Anatomy

Three parts in a single rounded shell — decrement, value, increment. Optional prefix/suffix slot for unit context.

days
  1. DecrementAlways on the left. Hit target 36×40px (default size). Greys out at min.
  2. ValueMono, centered, editable. Type-in is allowed — buttons are accelerators, not the only path.
  3. Suffix / prefixOptional unit slot — currency code, %, days. Sits flush against the value with a hairline divider.
  4. IncrementAlways on the right. Greys out at max.

07Don'ts

Number Input for IDs / large numbersIf a user can't reasonably reach the value with ±1 clicks, use a plain number field.

Unit baked into the valueAlways use the suffix/prefix slot. The input must be a parseable number.

Read-only input but enabled buttonsType-in is part of the contract. If the input is locked, lock the buttons too.

Recoloring buttons+/− are neutral. Don't tint them green/red — direction isn't a positive/negative judgment.

days

Unit floating outsideUse the built-in prefix/suffix slot — the unit belongs inside the rounded shell, not as separate text.

Custom button glyphsAlways minus / plus. Chevrons read as "open dropdown" or "next/prev", not increment.
