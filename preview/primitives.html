Design system

Primitives

Five small atoms that show up everywhere — Avatar, Money, Stat, Divider, Pill. Each is a thin wrapper over tokens; together they account for ~60% of all UI surface area in Cabiros. Get these right and every screen inherits the polish. (Looking for Tabs? It moved to its own component page.)

01Avatar

A small round container for an image, initials, or an icon. Used in lists, comments, presence indicators, and counterparty references throughout Cabiros.

Sizes .av-xs · .av-sm · .av-md · .av-lg · .av-xl

20 / 28 / 36 / 48 / 64 px. md is default — used in tables and lists. lg/xl for profile headers and detail panes only.

JD JD JD JD JD

Tones .av-{brand|success|warning|danger|info|neutral}

Tone signals what the avatar represents — brand for users, semantic tones for entities tied to a status (e.g. a counterparty pinned to "verified" reads green; a flagged one reads red). Do not pick tone for visual variety; let data drive it.

02Money

Currency display with tabular numbers, paired currency code, and consistent baseline alignment. Always mono — every digit is the same width so amounts in a column line up regardless of value.

Sizes .money-{sm|md|lg|xl|display}

Use md in tables, lg in card heroes, xl in detail summaries, display only for one big number per page (e.g. the wire amount on a confirmation modal).

Sentiment

Use a tone class only when the sign carries semantic weight — credits (incoming) green, debits (outgoing) red, projections muted. Do not color every transfer green just because it succeeded; success is conveyed by the row's status pill, not the amount.

03Stat

A KPI tile — three lines: label, value, delta. The whole grammar of a Cabiros dashboard is built from these. Stats appear in rows of 2–4; never alone (a single stat reads as decoration).

Anatomy

Mono uppercase label → big tabular value → optional signed delta with arrow. Optional foot line for context (timeframe, comparison).

Volume · 30d BRL 4.81M 12.4% vs. previous 30 days
Counterparties 128 flat no change this week
Failed transfers 2 0.4% 2 of 487 settlements

Compact .stat-compact

Borderless, transparent — for sidebars or page headers where the row already lives inside a card.

Pending 12
Settled · 24h 98
Avg. settle 2.1s
Live rails 7

Delta direction is sentiment, not value

"Failed transfers ↓" is good — fewer failures. So the delta arrow points down but the tone is success-leaning. Choose the tone class by what the change means, not by the sign of the number.

Errors · today 2 ↓ 0.4% · good
Latency p99 2.4s ↑ 18% · bad

05Pill

A compact, rounded badge that names a status, category, or metadata. Used in tables, cards, breadcrumbs, drawer summaries — anywhere a one-word label needs to read as data, not copy.

Tones .pill.{success|warning|danger|info|neutral|brand}

Tone is semantic — pick by the meaning of the status, never by visual variety. Tied to the same feedback color tokens that drive Toasts and Empty States, so a "Failed" pill reads identical wherever it appears.

Settled Pending Failed Processing Draft Live

Sizes .pill.sm · .pill (default) · .pill.lg

sm in dense tables, default in cards and headers, lg reserved for hero contexts where the pill is a primary signal (e.g. "Live · 7 rails" badge).

Decorators

Three options: dot (default — neutral, calm), icon (more visual weight, used in dashboards), bare text (most minimal — for category labels, not statuses). Pick one per context and stay consistent.

06Don'ts

Easy ways to break the rhythm these primitives create.

JD

Don't invent avatar tones. Stick to the six tokenised tones. Custom hex breaks the link between an avatar and the entity's status pill.

BRL 125.400,00

Don't render money in sans. Sans loses tabular alignment — digits drift across rows. Always wrap in .money.

Volume 4.81M 12.4%

Don't ship a single Stat. Stats are comparative. One alone reads as decoration; ship in rows of 2–4.

Section A
Section B

Don't double dividers. One line is enough. Double dividers signal "this card has a bug," not "this is more separated."

Settled successfully

Don't write sentences in pills. Pills are tags, not microcopy. One word, two max. Use Toast for transient sentences.

07Tokens

Every property of every primitive resolves to a token. Override at the token layer to retheme; never edit primitive CSS directly.
