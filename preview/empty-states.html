Design system

Empty states

The zero-data surface. When a page, section, or list has nothing to show, the empty state replaces it with an illustration, a one-line headline, optional copy, and a primary call-to-action that gets the user past the void.

01Overview

An empty state is the most important screen in a feature you've never used. It's the user's first impression of what something does and why they should care. Six illustration kinds — counterparties, receivables, transactions, search, error, default — cover every zero-data scenario in ATTRUS.

02Anatomy

Every empty state stacks the same five pieces vertically, center-aligned: illustration, title, description, action row (one or two buttons). The optional tone wraps the whole composition in a colored panel.

09Do & Don't

Empty states fail the same way over and over. Avoid these.

Do

  • State what's missing in plain language ("No counterparties yet").
  • Tell the user the next step in the description.
  • Use one primary action — the most-likely path forward.
  • Pick the illustration kind that matches the data type.
  • Use compact inside cards and drawers.

Don't

  • Don't leave a section blank — every empty state is a chance to teach.
  • Don't use a clever or jokey headline ("Oops, looks empty here!").
  • Don't stack three or more buttons — pick the one most-likely action.
  • Don't use the error illustration for empty data — only for actual errors.
  • Don't use the warning tone for normal empty states — reserve it for problems.
