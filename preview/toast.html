Design system

Toast

Ephemeral, non-blocking feedback that confirms the result of an action. Auto-dismisses after a short timeout, stacks vertically, never interrupts the user's flow.

01Anatomy

A toast is a short, single-purpose surface. Every part is optional except the title — the simplest valid toast is one line of text.

Transfer scheduled

$ 12,400.00 to Banco Itaú will settle on Apr 18.

  1. Leading iconSemantic glyph reinforces the variant. Match icon to severity (check / alert-triangle / x-circle / info).
  2. TitleOne line, sentence case, action-oriented. Required.
  3. DescriptionOptional second line for context, IDs, amounts. Stays under 2 lines.
  4. ActionOptional inline link — Undo, View, Retry. Single button only.
  5. CloseAlways present. Lets users dismiss before timeout.
  6. Progress barVisual countdown to auto-dismiss. Hides when duration: 0.

02Variants

Six semantic variants. Default for plain confirmations, four feedback variants matched to severity, plus brand for product announcements.

Default.toast

Neutral surface. Use when the message is informational and doesn't carry severity (e.g. "Settings saved").

Success.toast-success

Confirms an action completed as expected — payment sent, document uploaded, user invited.

Warning.toast-warning

Action succeeded but with a caveat — partial sync, near-limit threshold, deprecated path.

Danger.toast-danger

Action failed or a destructive event happened. Always offers a recovery path (Retry, Undo, View error).

Insufficient balance in CHL-001. Required: $ 24,800.00.

Info.toast-info

System-initiated update. The user didn't ask but should know — a sync started, a new feature is live.

Brand.toast-brand

Product announcements — new features, updates, tour entry points. Reserved for non-feedback messages.

03Positions

Six anchor points. The same app should pick one position and stick to it — moving toasts around teaches users to ignore them. ATTRUS default: top-right.

Anchor map

Top-right is the default for desktop dashboards. Bottom-center for mobile (closer to thumb, doesn't block status bars). Top/bottom-center for global system events that should command attention.

PositionClassWhen to use
Top-right (default).toast-stack-trDesktop dashboards — out of the way, predictable.
Top-center.toast-stack-tcGlobal system events (maintenance, sync) that need to be seen.
Top-left.toast-stack-tlRTL layouts, or when the right side hosts persistent panels.
Bottom-right.toast-stack-brApps with a prominent right rail (chat, properties).
Bottom-center.toast-stack-bcMobile-first surfaces — closer to the thumb.
Bottom-left.toast-stack-blRare. Only when bottom-right is occupied by primary controls.

04Duration & dismissal

Toasts auto-dismiss to stay non-blocking. Duration scales with severity — danger and actionable toasts stay longer.

Timing matrix

Hard rules: any toast with an action must stay long enough to read it (≥ 5s). Danger never auto-dismisses if it has an action — the user must decide.

VariantDefaultWith actionNotes
Default / Brand4s6sPure confirmation — fastest dismissal.
Success4s6sAction shouldn't punish a successful path.
Info5s7sSlightly longer — system-initiated, user wasn't expecting it.
Warning6s8sCaveat needs to be read.
Danger8sSticky (no auto-dismiss)Failure recovery — never time out a Retry button.

Pause-on-hover

The countdown pauses while the cursor is over the toast and resumes on leave. Lets users finish reading before dismissal kicks in.

Hover this toast

The progress bar will freeze until you move away.

05Actions

A toast can carry one inline action. More than one and it should become a Modal or an Inline Banner instead.

Recovery actions

Pair every danger and warning toast with a one-tap recovery — Retry, Undo, View. Verb-only labels, no "Click here".

Counterparty archived

Banco Itaú · BR moved to archive.

Connection lost

06Inverse

08Don'ts

Common toast pitfalls that erode trust or block the user.

Are you sure you want to delete?

This action is permanent.

Asking for decisionsToasts confirm, they don't ask. Anything destructive or with multiple options belongs in a Modal.

Saved. Here is some context that goes on for several lines and starts to look like a paragraph because the writer wasn't ruthless about cutting words and that's a problem because toasts are read in passing not studied.

Long copyTitle ≤ 1 line, description ≤ 2 lines. If you need more, you need a different surface.

Saved

Multiple actionsOne action only. Multiple verbs scatter attention and inflate the timeout. Promote to a Banner or Modal.

Stacking duplicatesIf the same toast fires repeatedly, debounce it or replace the existing one in place. Never stack identical messages.
